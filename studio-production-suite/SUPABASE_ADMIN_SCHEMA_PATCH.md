# Supabase SQL Patch (Copy/Paste)

Run this once in Supabase SQL Editor for admin content tools:

```sql
-- 1) Ensure admin users table exists (owner/admin account manager)
create table if not exists public.admin_users (
  id bigint generated by default as identity primary key,
  username text not null unique,
  display_name text,
  password_hash text not null,
  is_enabled boolean not null default true,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create index if not exists admin_users_enabled_idx on public.admin_users (is_enabled);

-- 2) Add podcast topic support (for admin grouping + topic creation)
alter table public.podcast_episodes
  add column if not exists topic text;

create index if not exists podcast_episodes_topic_idx
  on public.podcast_episodes (topic);

-- 2b) Podcast profiles (show-level) + episode relation
create table if not exists public.podcasts (
  id bigint generated by default as identity primary key,
  title text not null,
  slug text not null unique,
  hosts text,
  topic text,
  summary text,
  description text,
  cover_image_url text,
  is_published boolean not null default true,
  sort_order integer not null default 0,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

alter table public.podcast_episodes
  add column if not exists podcast_id bigint references public.podcasts(id) on delete cascade,
  add column if not exists sort_order integer not null default 0;

create index if not exists podcasts_admin_sort_idx
  on public.podcasts (topic, sort_order, title);

create index if not exists podcast_episodes_podcast_sort_idx
  on public.podcast_episodes (podcast_id, sort_order, published_at desc, id desc);

-- 3) Optional: improve band page/genre filtering performance
create index if not exists bands_admin_page_genre_idx
  on public.bands (era, genre, sort_order, name);

-- 4) Owner homepage controls (front-page card images)
alter table public.site_profiles
  add column if not exists home_legends_card_image_url text,
  add column if not exists home_scene_card_image_url text,
  add column if not exists home_podcast_card_image_url text,
  add column if not exists home_hub_card_image_url text,
  add column if not exists home_contact_card_image_url text;

-- 5) Owner tracks sorting controls
alter table public.tracks
  add column if not exists sort_order integer not null default 0;

create index if not exists tracks_admin_sort_idx
  on public.tracks (sort_order, release_date desc, id desc);
```

## Notes

- `bands.genre` already exists; admins can create a new genre by typing it in the admin form.
- `podcast_episodes.topic` is used for topic grouping in admin podcast manager.
- `podcasts` stores podcast profile records (name, hosts, about).
- `podcast_episodes` stores 1-2 latest episodes per podcast profile via `podcast_id`.
- `tracks.is_featured` is used as the "show on home player" toggle.
- No public-page CRUD controls are added; admin only.
